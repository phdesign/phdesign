Proxying large files through Web API

Here’s a few different implementations for proxying a large (1GB+) file through a Web API end point and onto another API.

Before you do anything, you need to update the web.config to support large file requests, it’s good practice to limit this to just the controllers / actions that you want to open up large file requests to, e.g.

<location path="api/learning/activities/uploadscorm">
  <system.web>
    <httpRuntime maxRequestLength="2097151" executionTimeout="360000"/>
  </system.web>
  <system.webServer>
    <security>
      <requestFiltering>
        <requestLimits maxAllowedContentLength="2147482624" />
      </requestFiltering>
    </security>
  </system.webServer>
</location>

We’ll need somewhere to post to, a great tool for this is requestb.in, you can anonymously create temporary endpoints to send data to and inspect the request.
